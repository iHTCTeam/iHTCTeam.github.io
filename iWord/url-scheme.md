<section lang="en" markdown="1">

iWord supports **`iword://`** shortcut links.

## Link format

```
iword://{host}?{params}
```

## Learning

| URL | Opens |
|-----|-------|
| `iword://daily` | Daily word |
| `iword://random` | Random entry |
| `iword://review` | SRS review session |
| `iword://learn` | Learn center |

## Browse & search

| URL | Opens |
|-----|-------|
| `iword://browse` | Browse all modules |
| `iword://browse?type=idiom` | Browse idioms |
| `iword://browse?type=xiehouyu` | Browse xiehouyu |
| `iword://entry?type=idiom&id=1` | Specific entry |
| `iword://search` | Search |
| `iword://favorites` | Favorites |

## Navigation & guides

| URL | Opens |
|-----|-------|
| `iword://home` | Home |
| `iword://settings` | Settings |
| `iword://widget-guide` | Widget guide |
| `iword://shortcuts-guide` | Shortcuts guide |
| `iword://usage-guide` | Usage guide |
| `iword://feature-guide` | Feature guide |
| `iword://feedback` | Feedback email |
| `iword://in-app-review` | In-app App Store rating |

### Entry types for `browse` and `entry`

`idiom` · `xiehouyu` · `tongue_twister` · `proverb` · `couplet` · `riddle`

</section>
